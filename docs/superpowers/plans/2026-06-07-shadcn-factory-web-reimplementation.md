**Goal:** Reimplement `apps/web` as a behavior-preserving full-route UI rewrite using factory-default shadcn components from preset `b5deKXWz4`.

**Architecture:** `packages/ui` becomes the generated, refreshable shadcn primitive package. `apps/web` keeps route behavior, hooks, API wrappers, fixtures, and workflow state, but rewrites visual composition to use shadcn primitives directly. Custom app UI survives only under grouped `apps/web/components/ui/*` exception folders with documentation.

**Tech Stack:** Next.js App Router, React 19, Tailwind CSS v4, shadcn registry `@shadcn`, `@cognify/ui`, next-themes, TanStack Query, TanStack Table, MSW, Vitest, Playwright.

Create `docs/04-engineering/standards/shadcn-factory-ui.md`:

```md
# Shadcn Factory UI Standard

## Rule

`packages/ui` contains factory-default shadcn primitives only. It must be safe to refresh from shadcn preset `b5deKXWz4` without losing Cognify product behavior.

`apps/web` composes those primitives directly in route and feature files. Custom reusable UI is allowed only when shadcn has no practical equivalent, and it must live under grouped folders in `apps/web/components/ui`.

## Package Boundary

- `packages/ui/src/components`, `packages/ui/src/hooks`, and `packages/ui/src/lib` are shadcn-managed.
- `packages/ui` must not contain Cognify copy, route behavior, procurement workflow state, or app shell decisions.
- `apps/web/components/ui` is the only reusable custom UI exception layer.
- Feature hooks, API wrappers, schemas, fixtures, and view-model mappers remain in `apps/web/features/*`.

## Allowed Custom UI Exception Groups

- `headers`: page headers, decorated titles, route mastheads.
- `graph`: graph, chart, network, and relationship visual composites not covered by shadcn `Chart`.
- `scorecard`: procurement score matrices and evaluation layouts that exceed shadcn `Table` or `Chart`.
- `procurement-table`: dense TanStack Table plus shadcn `Table` glue.
- `workflow-state`: workflow state summaries that have no direct shadcn primitive equivalent.

Each custom component file must include a top-level comment:

```ts
// shadcn-factory-exception: <reason>; primitives=<PrimitiveA, PrimitiveB>; routes=<route or feature>
```

## Required Replacements

- Use shadcn `Button` or `buttonVariants` instead of raw styled buttons.
- Use shadcn `Input`, `Textarea`, `Select`, `NativeSelect`, `Checkbox`, `RadioGroup`, `Switch`, `Field`, and `Form` instead of raw styled controls.
- Use shadcn `Table` primitives for tables.
- Use shadcn `Dialog`, `AlertDialog`, `Sheet`, `Popover`, `DropdownMenu`, `Command`, `Tabs`, `Tooltip`, `Calendar`, and `ScrollArea` for interactive surfaces.
- Use shadcn `Skeleton`, `Spinner`, `Empty`, `Alert`, and `Sonner` for loading, empty, error, and toast states.

## Theme

- Use the default shadcn theme generated by preset `b5deKXWz4`.
- Do not add route-level raw color palettes.
- Use token classes such as `bg-background`, `text-foreground`, `bg-card`, `text-muted-foreground`, `border-border`, and shadcn variants.

## Dark Mode

- Provide a visible app-shell theme toggle.
- Provide a plain `d` keyboard shortcut.
- The shortcut must not fire while typing in editable controls.
- The shortcut must not update the server-side profile theme setting.

## Review Checklist

- Focus rings are visible.
- Dialogs and sheets use shadcn/Radix focus management.
- Touch targets for workflow actions are at least 44px high.
- Mobile screens do not horizontally overflow.
- Form labels and errors are associated with controls.
- Data-heavy procurement screens remain scannable.
```

- [ ] **Step 2: Create the audit script**

Create `tooling/scripts/audit-shadcn-factory-ui.mjs`:

```js
import { readFileSync, readdirSync, statSync } from "node:fs";
import { join, relative } from "node:path";

const root = process.cwd();
const scanRoots = ["apps/web/app", "apps/web/components", "apps/web/features"];
const allowedCustomUiRoot = "apps/web/components/ui";
const rawInteractivePattern = /<(button|input|select|textarea)\b(?![^>]*data-allow-raw-control)/g;
const rawTablePattern = /<(table|thead|tbody|tr|th|td)\b(?![^>]*data-allow-raw-table)/g;
const roleDialogPattern = /role=["']dialog["']/g;
const customUiPattern = /^apps\/web\/components\/(?!ui\/|providers\/|shell\/shell-route-config|shell\/shell-types|shell\/shell-utils)/;
const packageUiForbiddenPattern = /Cognify|procurement|requisition|approval|rfq|quotation|tenant|vendor/i;
const packageUiRoots = ["packages/ui/src/components", "packages/ui/src/hooks", "packages/ui/src/lib"];

const failures = [];

function walk(dir) {
  let entries = [];
  try {
    entries = readdirSync(join(root, dir));
  } catch {
    return [];
  }

  return entries.flatMap((entry) => {
    const path = join(dir, entry);
    const absolute = join(root, path);
    const stat = statSync(absolute);
    if (stat.isDirectory()) return walk(path);
    if (!/\.(tsx|ts)$/.test(path)) return [];
    if (path.endsWith(".test.ts") || path.endsWith(".test.tsx")) return [];
    return [path];
  });
}

function assertNoMatches(file, pattern, message) {
  const source = readFileSync(join(root, file), "utf8");
  const matches = source.match(pattern);
  if (matches) {
    failures.push(`${file}: ${message} (${matches.length} match${matches.length === 1 ? "" : "es"})`);
  }
}

for (const file of scanRoots.flatMap(walk)) {
  assertNoMatches(file, rawInteractivePattern, "raw interactive control; use shadcn primitive");
  assertNoMatches(file, rawTablePattern, "raw table markup; use shadcn Table primitives");
  assertNoMatches(file, roleDialogPattern, "hand-rolled dialog role; use shadcn Dialog/AlertDialog/Sheet");

  if (customUiPattern.test(file) && !file.includes("/providers/")) {
    failures.push(`${file}: custom reusable UI outside ${allowedCustomUiRoot}`);
  }

  if (file.startsWith(`${allowedCustomUiRoot}/`) && !readFileSync(join(root, file), "utf8").includes("shadcn-factory-exception:")) {
    failures.push(`${file}: missing shadcn-factory-exception comment`);
  }
}

for (const file of packageUiRoots.flatMap(walk)) {
  assertNoMatches(file, packageUiForbiddenPattern, "Cognify-specific language in generated primitive package");
}

if (failures.length > 0) {
  console.error("Shadcn factory UI audit failed:");
  for (const failure of failures) console.error(`- ${failure}`);
  process.exit(1);
}

console.log("Shadcn factory UI audit passed.");
```

Create `apps/web/components/providers/theme-keyboard-shortcut.tsx`:

```tsx
"use client";

import { useTheme } from "next-themes";
import { useEffect } from "react";

function isEditableTarget(target: EventTarget | null): boolean {
  if (!(target instanceof HTMLElement)) return false;

  const tagName = target.tagName.toLowerCase();
  return (
    tagName === "input" ||
    tagName === "textarea" ||
    tagName === "select" ||
    target.isContentEditable
  );
}

export function ThemeKeyboardShortcut() {
  const { resolvedTheme, setTheme } = useTheme();

  useEffect(() => {
    function onKeyDown(event: KeyboardEvent) {
      if (event.key.toLowerCase() !== "d") return;
      if (event.metaKey || event.ctrlKey || event.altKey || event.shiftKey) return;
      if (isEditableTarget(event.target)) return;

      event.preventDefault();
      setTheme(resolvedTheme === "dark" ? "light" : "dark");
    }

    window.addEventListener("keydown", onKeyDown);
    return () => window.removeEventListener("keydown", onKeyDown);
  }, [resolvedTheme, setTheme]);

  return null;
}
```

import { Moon, Sun } from "lucide-react";
import { useTheme } from "next-themes";
import { Button, Kbd, Tooltip, TooltipContent, TooltipTrigger } from "@cognify/ui";

export function ThemeToggle() {
  const { resolvedTheme, setTheme } = useTheme();
  const nextTheme = resolvedTheme === "dark" ? "light" : "dark";
  const label = `Switch to ${nextTheme} mode`;

  return (
    <Tooltip>
      <TooltipTrigger asChild>
        <Button
          aria-label={label}
          variant="ghost"
          size="icon"
          onClick={() => setTheme(nextTheme)}
        >
          <Sun className="size-4 dark:hidden" aria-hidden="true" />
          <Moon className="hidden size-4 dark:block" aria-hidden="true" />
        </Button>
      </TooltipTrigger>
      <TooltipContent>
        <span>{label}</span>
        <Kbd className="ml-2">d</Kbd>
      </TooltipContent>
    </Tooltip>
  );
}
