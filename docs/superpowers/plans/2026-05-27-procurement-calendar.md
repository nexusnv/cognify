# Procurement Calendar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement P1-35 as a read-only, tenant-safe procurement calendar over existing RFQ, approval, requisition, quotation, and PO handoff dates.

**Architecture:** Add a query-backed Reporting read model exposed through `GET /api/procurement-calendar/events`. The web app consumes the generated OpenAPI client through `apps/web/features/procurement-calendar`, renders month/week/agenda views, and links events back to source workflows. No calendar event table, manual reminders, external sync, or source workflow mutations are introduced.

**Tech Stack:** Laravel 12, Sanctum tenant middleware, Eloquent read queries, OpenAPI/Orval, Next.js App Router, TanStack Query, MSW, Vitest/Testing Library, shadcn/Radix primitives via `@cognify/ui`.

## Scope Boundaries

In scope:

- Read-only calendar endpoint and UI.
- RFQ response due, RFQ invitation response due, approval task due, requisition needed-by, PO handoff requested PO date, and quotation validity events.
- Disabled source metadata for vendor document expiry and contract renewal.
- Generated API client use.
- Shell navigation and command palette entry.

Out of scope:

- Manual reminders.
- Persisted `procurement_calendar_events`.
- Editing source dates from the calendar.
- ICS/export/sync.
- Contract lifecycle or real vendor document expiry implementation.
- Notification or audit events from reading the calendar.

<?php

namespace Domains\Reporting\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class ListProcurementCalendarEventsRequest extends FormRequest
{
    public const SOURCE_TYPES = [
        'rfqDeadline',
        'approvalDue',
        'requisitionNeededBy',
        'poHandoff',
        'quotationValidity',
        'vendorDocumentExpiry',
        'contractRenewal',
    ];

    public const STATUSES = ['overdue', 'dueSoon', 'scheduled', 'completed', 'informational'];

    public function authorize(): bool
    {
        return true;
    }

    public function rules(): array
    {
        return [
            'from' => ['required', 'date'],
            'to' => ['required', 'date', 'after_or_equal:from'],
            'view' => ['sometimes', Rule::in(['month', 'week', 'agenda'])],
            'sourceTypes' => ['sometimes', 'array'],
            'sourceTypes.*' => ['string', Rule::in(self::SOURCE_TYPES)],
            'statuses' => ['sometimes', 'array'],
            'statuses.*' => ['string', Rule::in(self::STATUSES)],
            'q' => ['sometimes', 'nullable', 'string', 'max:120'],
            'limit' => ['sometimes', 'integer', 'min:1', 'max:500'],
        ];
    }

    public function withValidator($validator): void
    {
        $validator->after(function ($validator): void {
            if (!$this->filled(['from', 'to'])) {
                return;
            }

            $from = \Carbon\CarbonImmutable::parse($this->string('from'));
            $to = \Carbon\CarbonImmutable::parse($this->string('to'));

            if ($from->diffInDays($to) > 120) {
                $validator->errors()->add('to', 'The calendar range may not exceed 120 days.');
            }
        });
    }

    public function filters(): array
    {
        return [
            'view' => $this->input('view', 'month'),
            'sourceTypes' => $this->input('sourceTypes', []),
            'statuses' => $this->input('statuses', []),
            'q' => trim((string) $this->input('q', '')),
            'limit' => (int) $this->input('limit', 500),
        ];
    }
}

```php
private function statusFor(\Carbon\CarbonInterface $date, bool $open = true, bool $informational = false): string
{
    if ($informational) return 'informational';
    if (!$open) return 'completed';
    if ($date->isPast()) return 'overdue';
    if ($date->lessThanOrEqualTo(now()->addDays(7))) return 'dueSoon';
    return 'scheduled';
}

import type { ProcurementCalendarEvent } from "@cognify/api-client/schemas";

export function toDateKey(value: string): string {
  return value.slice(0, 10);
}

export function groupEventsByDate(events: ProcurementCalendarEvent[]) {
  return events.reduce<Record<string, ProcurementCalendarEvent[]>>((groups, event) => {
    const key = toDateKey(event.startsAt);
    groups[key] = [...(groups[key] ?? []), event];
    return groups;
  }, {});
}

export function sortEvents(events: ProcurementCalendarEvent[]) {
  return [...events].sort((a, b) => a.startsAt.localeCompare(b.startsAt) || a.title.localeCompare(b.title));
}

export function currentMonthRange(today = new Date()) {
  const from = new Date(today.getFullYear(), today.getMonth(), 1);
  const to = new Date(today.getFullYear(), today.getMonth() + 1, 0);
  return { from: from.toISOString().slice(0, 10), to: to.toISOString().slice(0, 10) };
}
```
