# Shadcn Factory UI Standard

## Rule

`packages/ui` contains factory-default shadcn primitives only. It must be safe to refresh from shadcn preset `b5deKXWz4` without losing Cognify product behavior.

`apps/web` composes those primitives directly in route and feature files. Custom reusable UI is allowed only when shadcn has no practical equivalent, and it must live under grouped folders in `apps/web/components/ui`.

## Package Boundary

- `packages/ui/src/components`, `packages/ui/src/hooks`, and `packages/ui/src/lib` are shadcn-managed.
- `packages/ui` must not contain Cognify copy, route behavior, procurement workflow state, or app shell decisions.
- `apps/web/components/ui` is the only reusable custom UI exception layer.
- Feature hooks, API wrappers, schemas, fixtures, and view-model mappers remain in `apps/web/features/*`.

## Allowed Custom UI Exception Groups

- `headers`: page headers, decorated titles, route mastheads.
- `graph`: graph, chart, network, and relationship visual composites not covered by shadcn `Chart`.
- `scorecard`: procurement score matrices and evaluation layouts that exceed shadcn `Table` or `Chart`.
- `procurement-table`: dense TanStack Table plus shadcn `Table` glue.
- `workflow-state`: workflow state summaries that have no direct shadcn primitive equivalent.

Each custom component file must include a top-level comment:

```ts
// shadcn-factory-exception: <reason>; primitives=<PrimitiveA, PrimitiveB>; routes=<route or feature>
```

## Required Replacements

- Use shadcn `Button` or `buttonVariants` instead of raw styled buttons.
- Use shadcn `Input`, `Textarea`, `Select`, `NativeSelect`, `Checkbox`, `RadioGroup`, `Switch`, `Field`, and `Form` instead of raw styled controls.
- Use shadcn `Table` primitives for tables.
- Use shadcn `Dialog`, `AlertDialog`, `Sheet`, `Popover`, `DropdownMenu`, `Command`, `Tabs`, `Tooltip`, `Calendar`, and `ScrollArea` for interactive surfaces.
- Use shadcn `Skeleton`, `Spinner`, `Empty`, `Alert`, and `Sonner` for loading, empty, error, and toast states.

## Theme

- Use the default shadcn theme generated by preset `b5deKXWz4`.
- Do not add route-level raw color palettes.
- Use token classes such as `bg-background`, `text-foreground`, `bg-card`, `text-muted-foreground`, `border-border`, and shadcn variants.

## Dark Mode

- Provide a visible app-shell theme toggle.
- Provide a plain `d` keyboard shortcut.
- The shortcut must not fire while typing in editable controls.
- The shortcut must not update the server-side profile theme setting.

## Review Checklist

- Focus rings are visible.
- Dialogs and sheets use shadcn/Radix focus management.
- Touch targets for workflow actions are at least 44px high.
- Mobile screens do not horizontally overflow.
- Form labels and errors are associated with controls.
- Data-heavy procurement screens remain scannable.
